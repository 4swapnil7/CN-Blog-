WELCOME TO Classful IP Addressing


IP address gives us the information about how to reach a host, not a part of our LAN.
It can be written in Dotted Decimal Notation, or Hexadecimal notation.
We're gonna learn here about the former.
(One example here.)

Classful addresses, of 32 bits, are divided in five sub classes
A, B, C, D ad E.

Now IPv4 address is divided in 2 parts.
Network ID
Host ID

Let's say a whole network/ organisation ,hospital has one network ID.
one hospital can have many devices, which are multiple individual hosts.
these hosts, here multiple users on computer or any network device, belong to this hospital network segment

CLASS A

Network length is of 8 bits.
host length of (32-8) 24 bits.
The first bit of the 8 bit long network Id is always 0, that defines the address as A class
So now remain 7 bits to distinguish various networks.
meaning 2^7=128 networks of which 2 are special.( 0.0.0.0 and 127.x.y.z)
Hence 2^24 -2=16,777,214 host ID are a part of each network.
Range of address: 1.x.x.x – 126.x.x.x
Default subnet mask: 255.x.x.x.
used by huge universities, companies like Google, and US military.

CLASS B

Network length is of 16 bits.
host length of (32-16) 16 bits.
The first TWO bitS of the 16 bit long network Id is always 10, that defines the address as B class
So now remain 14 bits to distinguish various networks.
meaning 2^14=16,384 networks
Hence 2^16 -2=16,777,214 host ID are a part of each network.
Range of address: 128.0.x.x – 191.255.x.x.
Default subnet mask: 255.255.x.x.
used by medium sized companies

CLASS C

Network length is of 24 bits.
host length of (32-24) 8 bits.
The first THREE bitS of the 24 bit long network Id is always 110, that defines the address as C class
So now remain 21 bits to distinguish various networks.
meaning 2^21=2097152 networks
Hence 2^8 -2=254 host ID are a part of each network.
Range of address: 192.0.0.x – 223.255.255.x.x.x.
Default subnet mask: 255.255.255.x.
used by small sized companies

CLASS D

No seggregation in prefix and suffix.
The first FOUR bits are always set 1110.
Interesting point is, remaining bits of address become what the hosts recognise.
Range of address: 224.0.0.0 – 239.255.255.255.
no subnet mask.
reserved for multi-casting

CLASS E

No seggregation in prefix and suffix.
The first FOUR bits are always set 1111.
Range of address: 240.0.0.0 – 255.255.255.254.
no subnet mask.
reserved for research purpose

REMEMBER:
(Not used network Id)
All bits set to 0 are not routed.
Starting with 127 as it's reserved for internal loop-back functions.
All bits set to 1 used as an IP broadcast address.

DISADVANTAGES
Classful addressing has become obsolete, since many addresses are wasted due to improper distribution.
Say class A, which has only 128 networks for whole world, but needs 16777214 hosts.
now only a handful of techgiants can be this large, hardly 20-30,
so remaining 100 times 16777214 address spaces are wasted.
Class C provides only 254 host addresses for one organization, not feasible even for small companies.
Class E addresses were mostly unused, so whole class space was wasted.
So the key issue is the rigid assignment of hosts addresses to sporadic number of networks.
